# Github 에 잘못 올라간 파일 삭제하기
.gitignore 파일을 생성하지 않고 push 를 진행하여 <u>node_modules 디렉터리가 올라가는 상황</u>이 발생했다. 이 때문에 검색을 하다보니 Github 에 잘못 올라간 파일을 삭제하는 방법이 있어서 정리한다.

# 🔴 원격 저장소에 push 된 파일 삭제
```bash
# remote 와 local 이렇게 양쪽 repository 에서 파일을 삭제한다.
$ git rm "파일 이름"

# remote 에서만 파일을 삭제한다.
$ git rm --cached "파일 이름"

# .idea 폴더 하위의 모든 파일을 삭제한다.
$ git rm --cached -r .idea/
```

나의 경우에는 remote 의 저장소에서만 파일을 삭제해야하므로 "git rm --cached" 명령어를 사용해야했고, 삭제의 대상이 node_modules 디렉터리였으므로 rm 의 옵션인 -r 을 추가하여 <u>"git rm --cached -r node_modules"</u> 명령어를 사용했다. 그 다음에 commit 과 push 를 수행해주면 끝난다.

## 🟠 rm 명령어의 옵션
리눅스 명령어인 `rm` 은 <u>파일이나 디렉터리를 삭제</u>할 때 사용하는 명령어이다. "<span class="mediumblue">-r</span>" 옵션을 붙이지 않으면 디렉터리는 삭제하지 못한다.

- -f(--force): 강제로 파일이나 디렉터리를 삭제
- -i(interactive): 삭제시마다 사용자에게 다시 질문
- -r(--recursive): 하위 디렉터리를 포함하여 모든 내용을 삭제
